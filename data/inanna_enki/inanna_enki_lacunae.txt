# Lacunae & indeterminacy registry — Inana and Enki (ETCSL 1.3.1)
#
# Purpose: tell the pipeline where high freedom is an ARTIFACT (damaged / unreadable
# text) rather than a FINDING (genuine interpretive ambiguity of intact myth).
#
# Key distinction (per the researcher's point):
#   Missing text is NOT automatically a problem. Most lacunae are harmless — the
#   surrounding myth still reads, and human + AI annotators converge on the function
#   anyway by restoring the invariant from context.
#   The problem is INDETERMINACY: a stretch where no cultural invariant exists to
#   read it, so no annotator (human or model) can assign a function — regardless of
#   whether text is missing. Damaged text is only part of it; the deeper issue is
#   that even the surviving lines have no established interpretation.
#
# Format (tab- or space-separated), one zone per line:
#   START-END   TYPE   note
#
# TYPE values:
#   missing        text is lost but the myth still reads around it; usually harmless,
#                  annotators still converge. Do NOT auto-flag freedom here as artifact
#                  unless it also lands on an indeterminate zone.
#   indeterminate  no cultural invariant to read this stretch; nobody (human/AI) can
#                  assign a function reliably. High freedom here is an ARTIFACT of
#                  unreadability, NOT genuine textual ambiguity. Exclude from claims
#                  about interpretive freedom; label forks here 'lacuna', not
#                  'text_ambiguous'.
#
# Lines beginning with # are comments. Blank lines ignored.
